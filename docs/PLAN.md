## 一、产品需求摘要（已确认）

### 1.1 定位

深度结构化歌词的美学重塑工具：精准检索 → 歌词清洗与分段 → 视觉排版 → 多比例图片导出。

### 1.2 业务边界

- **不提供**完整在线听歌；**不提供**试听（MVP 阶段）。
- **不提供**用户随意编辑原歌词文本；允许**隐藏行**、**金句选择**。
- **先不做**注册登录；**计划接入**按次付费，付完款自动下载高清图。

### 1.3 免费 / 付费策略

| 类型 | 预览 | 下载 |
|---|---|---|
| 免费用户 | 可预览 | 低分辨率 + 小水印 |
| 付费用户 | 同左 | 高清无水印 |

付费模式：**按次付费**（具体金额与支付渠道在接入支付时再定）。

### 1.4 数据源策略

- **MVP**：以网易云音乐非官方 API 为主（中文覆盖与元数据较全）。
- **架构**：抽象 `MusicSource` 接口，便于后续接入 Musixmatch、LrcLib 等。
- **后期**：扩展到更多数据源与风格。

### 1.5 终端与模板

- **终端**：响应式 Web（PC + 手机浏览器）。
- **首期模板**：经典小红书风格、日系手写风、中国风；后续按需扩展。
- **字体**：开源免费商用字体（思源系列、霞鹜系列、得意黑等）。

### 1.6 MVP 分期（已认可）

| 阶段 | 范围 |
|---|---|
| **P0** | 搜索锁定、歌词清洗、3 套模板、全文/金句、三比例导出、版权标注、响应式、免费低清+水印 / 付费高清（支付可 P0 末或 P1 初接入） |
| **P1** | 竖排、间距细调、动态背景与装饰、多语种切换、语义分段微调、支付闭环 |
| **P2** | 长图、A4/PDF、审美算法、更多模板、多数据源 |

## 二、技术架构

### 2.1 技术栈

| 层级 | 选型 | 说明 |
|---|---|---|
| 框架 | Next.js 14+ (App Router) | SSR、API Routes、部署友好 |
| UI | Tailwind CSS + shadcn/ui | 可定制组件 |
| 状态 | Zustand | 歌曲状态 + 编辑器状态 |
| 排版预览 | HTML + CSS | 利用 `writing-mode`、flex 等，竖排与复杂排版成本低 |
| 导出 | html-to-image | DOM 转位图，与预览一致 |
| 语言 | TypeScript | 全项目类型化 |

### 2.2 数据流

```
用户搜索
  → Next.js API Route（代理，隐藏第三方细节、可做限流）
  → 音乐数据源（MVP：网易云）
  → 返回列表

用户锁定歌曲
  → 获取歌词 + 元数据
  → 前端 lyric-parser（去 LRC 时间戳、整理空行与段落）
  → 写入 Zustand

用户调样式 / 换模板 / 选金句
  → editor-store 更新
  → 预览画布 DOM 实时重绘

用户导出
  → html-to-image 捕获画布节点
  → 免费：低 scale + 水印；付费：高 scale + 无水印（支付验证后放行）
```

### 2.3 数据源抽象层

```typescript
// 概念接口（实现见 lib/api/netease.ts 等）
interface MusicSource {
  search(keyword: string): Promise<SearchResult[]>;
  getSongDetail(id: string): Promise<SongDetail>;
  getLyric(id: string): Promise<RawLyric>;
}
```

### 2.4 模板系统（概念）

每套模板对应一组可序列化的样式配置（背景、字体、字号、行距、字距、对齐、方向、内边距、版权条样式等），便于「一键出图」与扩展。

### 2.5 导出策略（与 PRD 对齐）

| 维度 | 免费下载 | 付费下载 |
|---|---|---|
| 像素密度 | 较低（如 1× 基准分辨率） | 较高（如 3×） |
| 水印 | 有（小水印、品牌） | 无 |
| 格式 | PNG（可扩展 JPEG） | 同左 |

具体基准分辨率按画布设计尺寸在实现时写入 `lib/export.ts` 配置。

### 2.6 字体方案（首期）

| 用途 | 字体 | 适用模板 |
|---|---|---|
| 现代正文 | 思源黑体 Noto Sans CJK | 小红书等 |
| 标题强调 | 得意黑 Smiley Sans | 小红书 |
| 手写文艺 | 霞鹜文楷 / 霞鹜臻楷 | 日系、中国风 |
| 典雅正文 | 思源宋体 Noto Serif CJK | 中国风 |

字体文件建议放在 `public/fonts/`，通过 `@font-face` 或 Next 字体优化加载；大体积中文字体后续可做子集化优化。

## 四、页面与用户路径

1. **首页**：搜索 → 结果列表（封面、歌名、歌手、专辑、年份等）→ 点击确认进入编辑器。
2. **编辑器**：模板选择、全文/金句、字体与比例、实时预览画布、导出（免费 / 付费入口）。

**移动端**：优先保证搜索与预览可用；控制区可折叠或底部抽屉，画布区域足够大。

## 六、风险与备注

- **非官方音乐 API**：存在变更或限流风险；必须通过代理层封装，便于切换数据源。
- **歌词版权**：产品侧需在界面与导出物中规范展示来源与版权说明（已实现版权标注需求）。
- **支付无登录**：按次付费通常依赖订单号 + 支付回调生成**短期下载令牌**或**一次性下载链接**，具体方案在接入支付服务商时补充到本文档。
